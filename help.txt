Nice, this is a fun fit for your app. Let’s add **two simple Android screens** that simulate the web UI:

1. **Kochi Metro Dashboard**
2. **Embedded Documents**

Everything below is:

* Plain **XML layouts**
* Very simple **Kotlin**
* Uses only `ScrollView`, `LinearLayout`, `RecyclerView`, `CardView`
* Uses **dummy data** so it “just works” as a simulation.

You can wire these to your existing `LandingActivity` / `ChatActivity` however you like.

### 2.3 Kotlin – data + adapter + activity

**`DocumentItem.kt`**

```kotlin
package com.example.doculink

data class DocumentItem(
    val title: String,
    val tag: String,
    val description: String,
    val date: String,
    val size: String
)
```

**`DocumentAdapter.kt`**

```kotlin
package com.example.doculink

import android.view.LayoutInflater
import android.view.View
import android.view.ViewGroup
import android.widget.TextView
import androidx.recyclerview.widget.RecyclerView

class DocumentAdapter(
    private val items: List<DocumentItem>,
    private val onItemClick: (DocumentItem) -> Unit
) : RecyclerView.Adapter<DocumentAdapter.DocViewHolder>() {

    inner class DocViewHolder(view: View) : RecyclerView.ViewHolder(view) {
        val title: TextView = view.findViewById(R.id.txtDocTitle)
        val tag: TextView = view.findViewById(R.id.txtDocTag)
        val desc: TextView = view.findViewById(R.id.txtDocDescription)
        val meta: TextView = view.findViewById(R.id.txtDocMeta)
    }

    override fun onCreateViewHolder(parent: ViewGroup, viewType: Int): DocViewHolder {
        val v = LayoutInflater.from(parent.context)
            .inflate(R.layout.item_document_card, parent, false)
        return DocViewHolder(v)
    }

    override fun onBindViewHolder(holder: DocViewHolder, position: Int) {
        val item = items[position]
        holder.title.text = item.title
        holder.tag.text = item.tag
        holder.desc.text = item.description
        holder.meta.text = "${item.date} • ${item.size}"

        holder.itemView.setOnClickListener { onItemClick(item) }
    }

    override fun getItemCount(): Int = items.size
}
```

**`EmbeddedDocumentsActivity.kt`**

```kotlin
package com.example.doculink

import android.os.Bundle
import android.widget.Toast
import androidx.appcompat.app.AppCompatActivity
import androidx.recyclerview.widget.LinearLayoutManager
import androidx.recyclerview.widget.RecyclerView

class EmbeddedDocumentsActivity : AppCompatActivity() {

    private lateinit var recycler: RecyclerView

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_embedded_documents)

        recycler = findViewById(R.id.recyclerDocuments)
        recycler.layoutManager = LinearLayoutManager(this)

        val demoDocs = listOf(
            DocumentItem(
                "Financial Invoice - Traction Motor",
                "Financial",
                "Invoice for traction motor services and maintenance",
                "2025-01-15",
                "2.3 MB"
            ),
            DocumentItem(
                "Incident Report - Emergency Brake",
                "Safety",
                "Detailed incident report for emergency brake failure",
                "2025-01-14",
                "1.8 MB"
            ),
            DocumentItem(
                "Incident Report - Signal Failure",
                "Safety",
                "Signal system failure incident documentation",
                "2025-01-13",
                "2.1 MB"
            )
        )

        recycler.adapter = DocumentAdapter(demoDocs) { doc ->
            Toast.makeText(this, "Clicked: ${doc.title}", Toast.LENGTH_SHORT).show()
        }
    }
}
```

## 3. Wire it into your app

In `AndroidManifest.xml` add:

```xml
<activity android:name=".MetroDashboardActivity" />
<activity android:name=".EmbeddedDocumentsActivity" />
```

From any existing screen (e.g. `LandingActivity` or `ChatActivity`), you can open the Metro dashboard:

```kotlin
startActivity(Intent(this, MetroDashboardActivity::class.java))
```

---

This keeps everything **simple**, but visually close to your web UI:

* Light cards on soft grey background
* Clear hierarchy (big numbers, small labels)
* Document cards with titles, tags, description, meta info

Once you’re happy with the look, you can gradually hook these screens to your real backend instead of the dummy data.
